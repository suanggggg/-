# 用户故事（按优先级）

## 优先级说明
- P0: MVP 必须实现以便上线（核心流程可用）
- P1: 首批迭代应实现的增强功能
- P2: 后续优化与扩展功能

## P0（MVP）
1. 作为 学员，我可以注册/登录 系统并获得初始 1000 贡献分，以便开始使用基础功能。
2. 作为 管理员，我可以管理用户（增删改查）并查看操作日志，以便审计与权限控制。
3. 作为 管理员，我可以导入试题（Excel/CSV），以便快速构建试题库。
4. 作为 学员，我可以在笔试模块选择练习题并提交答案，系统展示即时评分与错题记录。
5. 作为 系统，我可以在试题库中进行全文检索并按科目/难度筛选题目。
6. 作为 学员，我可以参加基础人机面试训练（录音或录视频），并获得系统评分与雷达图展示。
7. 作为 专家，我可以对模拟测试进行线上评分并提交点评。
8. 作为 学员，我可以报名课程并支付（支持至少一种支付方式），系统能应用贡献分抵扣规则。
9. 作为 系统管理员，我可以查看贡献分账本并手动调整（异常处理）。

## P1（首批迭代）
1. 自动抓取外部题库并推送到待审核队列，支持来源记录与人工审核。
2. 专家可实时线上打分与适时打分，系统记录时间戳并统计一致性。
3. 无领导小组讨论的线上模拟功能（多人房间、计时、发言录制）。
4. 贡献分交易市场：发布邀约、确认交易、系统划转。
5. 学习成果每日两次测试并生成周/月排名与分享功能。

## P2（后续优化）
1. 行为与视频分析（目光、手势、面部表情）以辅助评分。
2. 高级题目智能推荐与错题自动生成练习集。
3. 高并发多人音视频优化与跨区域部署。
4. 更细粒度的权限与多组织支持（机构模式）。

---

每条用户故事应拆为若干验收标准（Acceptance Criteria），并在交付前由产品与 QA 共同确认。
